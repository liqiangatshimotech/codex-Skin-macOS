# 机器猫 Codex 主题安装说明（macOS）

这是一个给 Codex 桌面客户端使用的本地主题包。安装后会保留 Codex 原生侧栏、首页控件和输入框交互，并增加机器猫配色、顶部品牌栏与主题背景。

> 非 OpenAI 官方主题。当前安装器仅支持 macOS。

## 安装前准备

1. 确认官方客户端已经安装在下面任一位置：
   - `/Applications/ChatGPT.app`
   - `/Applications/Codex.app`
2. 在客户端中按 `Cmd + Q` 完整退出。只关闭窗口不等于退出。
3. 解压收到的 ZIP。不要把 `.command` 文件单独拿出来，必须保留整个文件夹结构。

## 一键安装

1. 双击 `安装机器猫主题.command`。
2. 如果 macOS 阻止打开，请右键该文件，选择“打开”，再确认一次。
3. 安装器会自动查找客户端和可用的 Node 运行时。
4. 如果 `/Applications` 没有写权限，终端会请求管理员密码。输入密码时屏幕不会显示字符，这是 macOS 的正常行为。
5. 看到 `Doraemon skin: installed` 后，按回车关闭终端。
6. 重新打开 Codex/ChatGPT 客户端。

安装器不会上传文件，不会修改模型、API Key 或 Base URL，也不会强制关闭正在运行的客户端。

## 恢复原版

1. 按 `Cmd + Q` 完整退出客户端。
2. 双击 `恢复原版界面.command`。
3. 看到恢复成功提示后重新打开客户端。

安装时会在客户端资源目录中创建原始文件备份：

```text
app.asar.codexskin.original.bak
```

恢复脚本会优先使用该备份。

## 检查状态

双击 `检查主题状态.command`。正常安装后的输出应包含：

```text
Backup: present
Doraemon skin: installed
```

## 客户端更新后主题消失

客户端自动更新可能替换 `app.asar`，主题随之消失。这不是数据丢失；再次双击 `安装机器猫主题.command` 即可。

## 常见问题

### 打开后仍是旧界面

确认使用 `Cmd + Q` 完整退出后再启动。仅关闭窗口不会重新加载资源。

### 页面空白或新版客户端布局异常

先运行 `恢复原版界面.command` 恢复官方界面。主题依赖客户端页面结构，官方大版本更新后可能需要新版主题包。

### 提示找不到客户端

请确认应用位于 `/Applications`，而不是“下载”目录或其他自定义位置。高级用户可在终端中设置路径后运行：

```bash
CODEX_APP_PATH="/自定义路径/Codex.app" ./安装机器猫主题.command
```

### 提示找不到 Node

安装器会优先使用客户端自带的 Node。如果当前客户端版本没有附带 Node，可先安装 Node.js，再重新运行安装器。

## 手动终端命令

在本文件夹中执行：

```bash
# 安装
./安装机器猫主题.command

# 检查
./检查主题状态.command

# 恢复
./恢复原版界面.command
```
